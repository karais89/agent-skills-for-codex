# 제품 요구사항 색인

`docs/product-specs/`는 제품 또는 기능 단위 요구사항을 보관한다. 새 요구사항은 `spec` 또는 `spec:` alias로 `$harness-product-spec`을 사용해 작성한다.

## 목록

| Spec | 상태 | 마지막 갱신 | 비고 |
| --- | --- | --- | --- |
| TBD | draft | TBD | 첫 spec 작성 시 이 행을 교체한다. |

## 상태 값

- `draft`: 요구사항을 정리 중이다.
- `ready`: 실행 계획으로 나눌 수 있다.
- `implemented`: 요구사항이 구현되고 검증되었다.
- `superseded`: 다른 spec으로 대체되었다.
