# 아키텍처

이 문서는 프로젝트의 장기 구조와 주요 설계 결정을 설명한다. 개별 기능 요구사항은 `docs/product-specs/`에, 구현 순서와 진행 상태는 `docs/exec-plans/`에 둔다.

## 시스템 개요

- 목적: 이 프로젝트가 해결하는 문제를 적는다.
- 주요 사용자: 대상 사용자와 운영자를 적는다.
- 실행 환경: 런타임, 배포 환경, 외부 의존성을 적는다.

## 데이터와 상태

- 저장소, 데이터베이스, 캐시, 외부 API를 적는다.
- 중요한 데이터 흐름과 소유권 경계를 적는다.
- 마이그레이션이나 호환성 제약이 있으면 적는다.

## 인터페이스

- 공개 API, CLI, UI entrypoint, background job을 적는다.
- 모듈 간 계약과 변경 시 주의할 점을 적는다.

## 품질 속성

- 보안: 인증, 인가, secret 관리, 입력 검증 기준을 적는다.
- 성능: 주요 latency, 처리량, 리소스 사용 기준을 적는다.
- 안정성: retry, timeout, rollback, monitoring 기준을 적는다.
- 접근성: UI가 있다면 키보드, 스크린 리더, 대비 기준을 적는다.
