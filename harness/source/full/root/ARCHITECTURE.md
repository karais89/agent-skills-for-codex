# 아키텍처

이 문서는 프로젝트의 장기 구조, 주요 구성요소, 설계 결정을 설명한다. 기능별 요구사항은 `docs/product-specs/`에, 구현 순서와 진행 상태는 `docs/exec-plans/`에 둔다.

## 작성 원칙

- 현재 구현과 다르게 쓰지 않는다. 아직 결정되지 않은 항목은 `TBD`로 남긴다.
- 기능 요구사항, 사용자 시나리오, 수용 기준은 이 문서가 아니라 product spec에 둔다.
- 작업 순서, 체크리스트, 검증 기록은 이 문서가 아니라 ExecPlan에 둔다.
- 모듈 경계, 데이터 소유권, 외부 인터페이스, 품질 기준이 바뀌면 이 문서를 갱신한다.

## 알려진 제약

- 현재 구조에서 의도적으로 감수하는 제약을 적는다.
- 후속 작업이 필요한 구조적 위험을 적는다.

## 갱신 기준

다음 변경이 생기면 이 문서를 함께 갱신한다.

- 새 런타임, 프레임워크, 데이터 저장소, 외부 서비스가 추가된다.
- 주요 모듈 책임이나 데이터 소유권이 바뀐다.
- 공개 API, CLI, UI entrypoint, background job 계약이 바뀐다.
- 보안, 성능, 안정성, 접근성 기준이 바뀐다.
- 중요한 설계 결정이 내려지거나 기존 결정이 폐기된다.
