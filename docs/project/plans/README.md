# 내부 실행 계획 기준

`docs/project/plans/`는 이 저장소 자체를 개발할 때 사용하는 실행 계획을 보관한다. 다른 프로젝트에 배포할 계획 문서 양식은 이곳이 아니라 `harness/templates/` 아래에 둔다.

## 사용 기준

- 복잡한 기능 구현, 큰 리팩터링, 장시간 작업은 작업 전에 실행 계획을 작성하거나 갱신한다.
- 진행 중인 계획은 `active/`, 완료된 계획은 `completed/` 아래에 둔다.
- 단순 문서 수정이나 작은 단일 파일 변경에는 과도한 계획 문서를 만들지 않는다.

## 산출물과의 구분

내부 계획은 이 저장소의 작업 상태를 기록하는 문서다. 반복해서 재사용할 수 있는 구조가 확인되면 대상 프로젝트 관점의 템플릿으로 다시 작성해 `harness/templates/`에 둔다.
