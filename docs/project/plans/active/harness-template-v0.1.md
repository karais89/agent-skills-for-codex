# 하네스 템플릿 v0.1 실행 계획

## 목적

이 문서는 `harness/templates/` 아래에 만들 하네스 템플릿 v0.1의 구현 범위, 진행 체크리스트, 검증 기준을 관리한다. v0.1의 목표는 다른 프로젝트에 복사해 바로 채울 수 있는 최소 문서 하네스 템플릿을 제공하는 것이다.

이 문서는 내부 실행 계획이며, 실제 배포 산출물은 아니다. 실제 템플릿 문서는 `harness/templates/` 아래에 별도로 작성한다.

## 현재 상태

- `harness/templates/README.md`에 템플릿 보관 위치와 작성 기준이 정리되어 있다.
- `docs/project/plans/README.md`는 진행 중인 내부 계획을 `active/`, 완료된 계획을 `completed/` 아래에 둔다고 정의한다.
- v0.1 템플릿 산출물은 아직 작성되지 않았다.
- 이번 계획은 실제 템플릿 구현 전, 작업 범위와 완료 기준을 고정하기 위해 작성한다.

## v0.1 범위

v0.1은 다른 프로젝트에 복사 가능한 최소 문서 하네스 템플릿을 만든다. 대상 프로젝트가 코드베이스 구조, 작업 계획, 검증 기준, 에이전트 작업 지침을 한곳에서 파악할 수 있도록 문서 뼈대를 제공한다.

v0.1에서 다루는 범위는 다음과 같다.

- 프로젝트 루트용 `AGENTS.md` 템플릿 작성
- 프로젝트 문서 진입점과 구조 설명 템플릿 작성
- 장시간 작업과 큰 변경을 위한 실행 계획 템플릿 작성
- 검증 기준을 기록하는 문서 템플릿 작성
- 템플릿 문서 간 링크와 공통 용어 정리
- v0.1 완료 기준과 검증 절차 정리

- [ ] `harness/templates/AGENTS.md` 초안을 작성한다.

- [ ] 각 템플릿에 적용 대상, 필수 입력, 기대 결과, 검증 방법을 반영한다.
- [ ] 문서 간 상대 링크가 실제 경로와 맞는지 확인한다.
- [ ] `하네스`, `템플릿`, `실행 계획`, `검증`, `대상 프로젝트` 같은 공통 용어를 일관되게 정리한다.
- [ ] v0.1에서 사용자가 반드시 채워야 하는 자리표시자를 확인한다.
- [ ] v0.1 완료 기준을 산출물 문서와 내부 계획 문서 양쪽에서 확인 가능하게 정리한다.
- [ ] 변경 범위가 `harness/templates/`와 필요한 내부 계획 갱신에 한정되는지 확인한다.

## v0.1 산출물 목록

- `harness/templates/AGENTS.md`

- `harness/templates/docs/validation.md`

## 제외 항목

다음 항목은 v0.1 실행 범위에 포함하지 않는다.

- v0.1 템플릿을 테스트 프로젝트나 fixture에 실제 적용하는 작업
- `codex exec` 기반 행동 검증 시뮬레이션
- 반복 작업을 자동화하는 스크립트 작성
- 실행 계획 작성, 검증 결과 해석, 템플릿 적용을 위한 스킬 작성
- phase/step 실행기나 큰 자동화 구조 설계
- UI 프로젝트 전용 브라우저 검증 기준 작성
- 내부 운영 문서 전체 리팩터링

## 검증 기준

v0.1 템플릿 구현이 끝나면 다음 기준으로 확인한다.

- `git diff --check`가 통과한다.
- v0.1 산출물 목록의 모든 파일이 존재한다.
- 각 템플릿 문서의 상대 링크가 실제 경로와 맞는다.
- `AGENTS.md` 템플릿이 대상 프로젝트의 짧은 진입점 역할에 집중한다.
- `docs/README.md` 템플릿이 대상 프로젝트 문서 구조를 안내한다.
- `docs/project-map.md` 템플릿이 코드와 문서 위치를 빠르게 찾게 한다.
- `docs/plans/exec-plan.md` 템플릿이 큰 작업의 목표, 범위, 체크리스트, 검증, 완료 조건을 기록하게 한다.
- `docs/validation.md` 템플릿이 테스트, 수동 확인, 남은 위험을 기록하게 한다.
- v0.1 제외 항목이 실제 산출물에 구현 범위처럼 섞이지 않는다.

## 완료 조건

다음 조건을 모두 만족하면 v0.1을 완료로 본다.

- v0.1 진행 체크리스트의 모든 항목이 완료되어 있다.
- v0.1 산출물 목록의 파일이 모두 작성되어 있다.
- 검증 기준을 실행하고 결과를 기록했다.
- 변경 범위에 v0.1과 무관한 문서, 코드, 설정 변경이 없다.
- 필요한 경우 `harness/templates/README.md`가 새 템플릿 목록을 반영한다.
- 변경 사항이 의미 있는 작업 단위로 커밋되어 있다.

## 다음 로드맵

후속 버전은 방향만 이 문서에 남긴다. 각 버전의 세부 실행 계획은 해당 단계에서 별도 계획 문서로 작성한다.

### v0.2: 템플릿 적용 시뮬레이션

- 테스트 프로젝트나 fixture에 v0.1 템플릿을 적용한다.
- `codex exec` 기반 행동 검증 절차를 문서화한다.
- 기대 행동과 실제 결과 차이를 기록한다.

### v0.3: 스킬 후보 추출

- 문서만으로 반복 수행이 불안정한 작업을 식별한다.
- 실행 계획 작성, 검증 결과 해석, 템플릿 적용 같은 스킬 후보를 정리한다.
- 스킬로 만들 가치가 있는 작업과 문서 지침으로 충분한 작업을 구분한다.

### v0.4: 선택적 자동화 검토

- 반복되는 적용 절차와 검증 절차만 스크립트화할지 검토한다.
- phase/step 실행기나 큰 자동화는 실제 반복 필요가 확인된 뒤 별도 계획에서 다룬다.
- 자동화가 문서 하네스의 이해 가능성을 낮추지 않는지 확인한다.

### v0.5: UI·브라우저 검증 보강

- UI 프로젝트용 검증 체크리스트를 추가한다.
- 브라우저 기반 확인 기준을 정리한다.
- 화면 확인 결과를 템플릿 문서에 기록하는 최소 형식을 검토한다.

## 완료 후 처리

v0.1 진행 체크리스트가 모두 완료되면 이 문서를 `docs/project/plans/completed/`로 이동한다. 이동 전이나 이동 직후 다음 내용을 결과 회고로 추가한다.

- 완료 날짜
- 실제 작성된 산출물 목록
- 실행한 검증 명령과 결과
- 계획과 달라진 점
- 남은 위험과 후속 작업

완료 후 변경 범위를 확인하고, v0.1 구현 단위의 커밋을 만든다.
