# 프로젝트 운영 문서 기준

`docs/project/`는 이 저장소 자체를 개발하고 운영할 때 에이전트가 따르는 내부 문서를 보관한다. 이 문서들은 하네스 산출물이 아니며, 다른 프로젝트에 그대로 복사하는 것을 기본으로 하지 않는다.

## 역할

- 이 저장소에서 작업하는 에이전트의 행동 원칙과 작업 규칙을 정리한다.
- 변경 범위, 검증, 커밋, 문서 관리처럼 현재 저장소 운영에 필요한 기준을 둔다.
- 하네스 산출물을 만들기 전에 판단 기준과 작업 방식을 안정화한다.

## 산출물과의 관계

`harness/`는 다른 프로젝트에 적용할 템플릿과 산출물을 보관한다. `docs/project/`의 규칙을 재사용할 가치가 확인되면, 내부 문서를 직접 옮기지 말고 대상 프로젝트 관점의 템플릿으로 다시 작성해 `harness/`에 둔다.

## 검증 방향

하네스 템플릿은 이 저장소 루트가 아니라 템플릿을 적용한 별도 테스트 대상에서 검증한다. 템플릿 적용 품질을 확인해야 할 때는 테스트 프로젝트나 fixture에서 `codex exec` 기반 시뮬레이션을 사용할 수 있도록 입력 프롬프트, 대상 파일, 기대 결과, 실행 결과를 문서화한다.
