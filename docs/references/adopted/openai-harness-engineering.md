# OpenAI Harness Engineering 참고

원문: https://openai.com/ko-KR/index/harness-engineering/

확인일: 2026-04-26

## 핵심 요약

OpenAI의 하네스 엔지니어링 문서는 Codex 같은 코딩 에이전트가 안정적으로 일할 수 있도록 저장소 구조, 문서, 테스트, 관측 가능성, 검토 루프를 설계하는 방식을 설명한다. 핵심은 사람이 코드를 직접 더 많이 쓰는 것이 아니라, 에이전트가 읽고 실행하고 검증할 수 있는 환경을 만드는 것이다.

## 프로젝트에 적용할 원칙

- `AGENTS.md`는 긴 백과사전이 아니라 짧은 진입점과 목차 역할을 하게 한다.
- 깊은 지식은 `docs/` 아래에 버전 관리되는 문서로 정리한다.
- 에이전트가 볼 수 없는 정보는 실질적으로 존재하지 않는 것으로 취급하고, 중요한 의사결정과 운영 지식을 저장소 안에 남긴다.
- 큰 작업은 작은 빌딩 블록으로 나누고, 각 단계가 검토와 검증을 거칠 수 있게 한다.
- 테스트, 린터, 스크립트, 관측 가능성 같은 피드백 루프를 에이전트가 직접 실행하고 해석할 수 있게 만든다.
- 아키텍처 취향은 구두 지시보다 기계적으로 검증 가능한 규칙으로 강제하는 편이 낫다.
- 문서는 작성 후 방치하지 않고 실제 코드 동작과 맞는지 주기적으로 정리한다.

## 참고할 부분

- 빈 Git 저장소에서 시작하더라도 초기 구조, CI, 포맷, 패키지 관리, `AGENTS.md`를 먼저 갖춘다.
- 저장소 지식은 `docs/`를 중심으로 분류하고 색인화한다.
- 복잡한 작업은 실행 계획과 의사결정 로그를 함께 버전 관리한다.
- UI, 로그, 메트릭, 추적 정보는 에이전트가 직접 읽을 수 있는 형태로 노출할수록 검증 품질이 올라간다.

## 이 프로젝트의 반영 상태

- 루트 `AGENTS.md`를 프로젝트 지침의 진입점으로 사용한다.
- 세부 참고 자료는 `docs/references/` 아래에서 채택 상태에 따라 분류해 보관한다.
- 외부 참고 저장소는 `reference-repos/adopted/`와 `reference-repos/not-adopted/`로 나누되 Git 추적에서는 제외한다.
